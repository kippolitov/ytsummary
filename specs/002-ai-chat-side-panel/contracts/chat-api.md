# Contract: Chat API

**Endpoint**: `POST /api/chat`
**Version**: v1
**Feature**: 002-ai-chat-side-panel

---

## Overview

The chat endpoint accepts a video transcript and conversation history, then streams an AI-generated reply back to the caller as a Server-Sent Events (SSE) text stream. It is the sole backend endpoint for this feature.

Authentication follows the same pattern as the existing `/api/analyze` endpoint: the Azure Function host key is passed as a `?code=` URL query parameter.

---

## Request

**Method**: `POST`
**Content-Type**: `application/json`
**Auth**: `?code=<function-key>` query parameter

### Body

```json
{
  "videoId": "dQw4w9WgXcY",
  "videoTitle": "Rick Astley - Never Gonna Give You Up",
  "transcript": "We're no strangers to love...",
  "messages": [
    {
      "role": "user",
      "content": "What is this video about?"
    },
    {
      "role": "assistant",
      "content": "This video is a classic 1987 pop song by Rick Astley..."
    },
    {
      "role": "user",
      "content": "What year was it released?"
    }
  ],
  "mode": "chat"
}
```

### Field Definitions

| Field | Type | Required | Constraints |
|-------|------|----------|-------------|
| `videoId` | string | Yes | 11-character YouTube video ID matching `/^[a-zA-Z0-9_-]{11}$/` |
| `videoTitle` | string | Yes | Max 500 characters |
| `transcript` | string | Yes | Max 80,000 characters; server rejects requests exceeding this limit with 422 |
| `messages` | array | Yes | 1–50 items; must include at least the most recent user message as the last item |
| `messages[].role` | `'user' \| 'assistant'` | Yes | |
| `messages[].content` | string | Yes | 1–2,000 characters for user messages; no limit for assistant messages |
| `mode` | `'chat' \| 'blog-post'` | No | Defaults to `'chat'` when omitted |

### Mode Semantics

- **`chat`**: The AI answers the final user message in the context of the transcript and prior conversation.
- **`blog-post`**: The AI ignores the conversation history and generates a full blog post (title, introduction, 2–5 sections, conclusion) from the video transcript. The final user message is ignored; the generation is entirely system-driven.

---

## Response (Success — HTTP 200)

**Content-Type**: `text/event-stream`
**Transfer-Encoding**: chunked

The response body is a stream of SSE events. Each event line follows the format:

```
data: <JSON-payload>\n\n
```

### Delta chunk

Emitted for each text fragment generated by the AI:

```
data: {"delta":"The video was released"}
```

```
data: {"delta":" in 1987."}
```

### Completion marker

Emitted once when generation is complete:

```
data: [DONE]
```

The client MUST treat receipt of `data: [DONE]` as the end of the stream and stop reading.

### Full example stream

```
data: {"delta":"The video was released"}

data: {"delta":" in 1987, specifically"}

data: {"delta":" on July 27th."}

data: [DONE]
```

---

## Error Responses

All error responses return JSON with `Content-Type: application/json`. No streaming occurs on error.

| HTTP Status | Error Code | Meaning | Retryable |
|-------------|------------|---------|-----------|
| `400` | `bad-request` | Missing required field or malformed JSON | No |
| `422` | `transcript-too-long` | Transcript exceeds 80,000 characters | No |
| `422` | `too-many-messages` | `messages` array exceeds 50 items | No |
| `429` | `rate-limited` | Azure OpenAI throttling | Yes |
| `500` | `service-error` | Unexpected server error | Yes |
| `503` | `service-unavailable` | Azure OpenAI unreachable | Yes |

### Error body shape

```json
{
  "error": {
    "code": "transcript-too-long",
    "message": "Transcript exceeds the 80,000 character limit."
  }
}
```

## CORS

The function MUST include `Access-Control-Allow-Origin: *` (or the extension's specific origin) and handle `OPTIONS` preflight requests, consistent with the existing `/api/analyze` configuration.

## System Prompt Behavior

The function constructs the model prompt as follows:

**For `mode: 'chat'`**:
```
System: You are an AI assistant helping the user understand a YouTube video.
        Video title: {videoTitle}
        Transcript:
        ---
        {transcript}
        ---
        Answer questions based primarily on the transcript. If a question cannot
        be answered from the transcript, say so clearly, then provide general context.
        Keep answers concise and accurate.

[conversation history messages]
[final user message]
```

**For `mode: 'blog-post'`**:
```
System: You are a professional content writer. Based on the following YouTube video
        transcript, write a well-structured blog post suitable for publication.
        Video title: {videoTitle}
        Transcript:
        ---
        {transcript}
        ---
        The blog post MUST include:
        - A compelling title
        - An introduction (2–3 sentences)
        - 2–5 content sections, each with a heading and body paragraphs
        - A conclusion (2–3 sentences)
        Target length: 600–1,200 words.
        Use markdown formatting (## for section headings, **bold** for emphasis).

User: Generate a blog post about this video.
```

---

## Non-functional Requirements

- **Timeout**: The function MUST begin streaming the first delta chunk within 10 seconds of receiving the request. If the model API does not respond within 10 seconds, the function returns HTTP 503.
- **Max generation time**: 60 seconds total for the full response (blog posts may be long).
- **Idempotency**: Not applicable — AI responses are non-deterministic.
